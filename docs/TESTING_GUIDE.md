## Prerequisites

### Required Software

- Postman v10 or later
- Java 17+
- Maven 3.8+

### Application Setup

1. **Clone and build:**
   ```bash
   git clone <repository-url>
   cd payment-gateway-service
   ./mvnw clean install
   ```

2. **Start the application:**
   ```bash
   ./mvnw spring-boot:run -Dspring-boot.run.profiles=local
   ```

3. **Verify health:**
   ```bash
   curl http://localhost:8080/actuator/health
   ```

## Setup

### Import Postman Collection

1. Open Postman
2. Click **Import**
3. Select files from `postman/` directory:
   - `Payment Gateway.postman_collection.json`
   - `Payment Gateway Local.postman_environment.json`

### Configure Environment

1. Select "Payment Gateway Local" environment
2. Click environment eye icon to view variables
3. Variables will auto-populate when running requests

## Test Scenarios

### 1. Merchant Lifecycle Tests

#### Test: Register New Merchant

**Request:** `1. Merchant Management` → `Register Merchant`

**Expected:**
- Status: 201 Created
- Response has merchant ID
- Merchant status is ACTIVE

**Test Script Validation:**
```javascript
pm.test("Status code is 201", () => pm.response.to.have.status(201));
pm.test("Merchant ID exists", () => {
    const json = pm.response.json();
    pm.expect(json.data.id).to.exist;
});
```

#### Test: Update Merchant

**Request:** `1. Merchant Management` → `Update Merchant`

**Expected:**
- Status: 200 OK
- Name/email/webhook updated

#### Test: Suspend Merchant

**Request:** `1. Merchant Management` → `Suspend Merchant`

**Expected:**
- Status: 200 OK
- Merchant status changed to SUSPENDED

### 3. Payment Processing Tests

#### Test: Process Valid Payment

**Request:** `3. Payment Processing` → `Process Payment`

**Headers:**
```
X-Idempotency-Key: <auto-generated>
```

**Body:**
```json
{
  "merchantId": "{{merchantId}}",
  "amountInCents": 10000,
  "currency": "USD",
  "customerId": "{{customerId}}",
  "description": "Test payment"
}
```

**Expected:**
- Status: 200 OK
- Payment ID generated
- Status: AUTHORIZED
- Idempotency key stored

#### Test: Idempotency

**Requests:** `7. Complete Flow` → Idempotency tests

**Steps:**
1. Run "IDEMPOTENCY TEST - First Request"
2. Note the payment ID
3. Run "IDEMPOTENCY TEST - Second Request (Same Key)"
4. Verify same payment ID returned

**Expected:**
- Both requests return identical payment ID
- No duplicate charge created

#### Test: Capture Payment

**Request:** `3. Payment Processing` → `Capture Payment`

**Expected:**
- Status: 200 OK
- Payment status: CAPTURED
- Transaction ID generated

#### Test: Cancel Payment

**Request:** `3. Payment Processing` → `Cancel Payment`

**Expected:**
- Status: 200 OK
- Payment status: CANCELLED

## Automated Testing

### Using Collection Runner

1. Open **Runner** in Postman
2. Select "Payment Gateway API" collection
3. Select "Payment Gateway Local" environment
4. Set iterations to 1
5. Click **Run Payment Gateway API**

### Running Specific Folders

To run only payment tests:
1. Open Runner
2. Expand collection
3. Check only "3. Payment Processing" folder
4. Run selected requests

### Viewing Results

After running:
- Green = Test passed
- Red = Test failed
- Click request to view detailed results

## CI/CD Integration

### Newman (Postman CLI)

Install Newman:
```bash
npm install -g newman
```

Run collection:
```bash
newman run postman/Payment\ Gateway.postman_collection.json \
  -e postman/Payment\ Gateway\ Local.postman_environment.json \
  --reporters cli,json \
  --reporter-json-export results.json
```

### Maven Integration

Add to `pom.xml`:
```xml
<plugin>
  <groupId>org.codehaus.mojo</groupId>
  <artifactId>exec-maven-plugin</artifactId>
  <version>3.1.0</version>
  <executions>
    <execution>
      <id>run-postman-tests</id>
      <phase>integration-test</phase>
      <goals>
        <goal>exec</goal>
      </goals>
      <configuration>
        <executable>newman</executable>
        <arguments>
          <argument>run</argument>
          <argument>postman/Payment Gateway.postman_collection.json</argument>
          <argument>-e</argument>
          <argument>postman/Payment Gateway Local.postman_environment.json</argument>
        </arguments>
      </configuration>
    </execution>
  </executions>
</plugin>
```

### GitHub Actions

```yaml
name: API Tests

on: [push, pull_request]

jobs:
  api-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Java
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Start Application
        run: ./mvnw spring-boot:run -Dspring-boot.run.profiles=local &

      - name: Wait for Application
        run: sleep 30

      - name: Install Newman
        run: npm install -g newman

      - name: Run Postman Tests
        run: |
          newman run postman/Payment\ Gateway.postman_collection.json \
            -e postman/Payment\ Gateway\ Local.postman_environment.json \
            --reporters cli
```

## Debugging Tips

### Enable Console Logging

View Postman Console:
```
View → Show Postman Console (Ctrl+Alt+C)
```

### Check Environment Variables

Click eye icon to verify:
- All IDs are populated
- No empty required variables

### Inspect Request/Response

1. Click request
2. View **Body** tab for request payload
3. View **Response** tab for server response

### Common Issues

| Issue | Solution |
|-------|----------|
| 404 Not Found | Verify app is running, check baseUrl |
| 400 Bad Request | Check required fields, verify IDs populated |
| Tests failing | Run requests in order, check console output |
